# Product Guidelines

## Communication Style
- **Prose:** Professional, concise, and direct. Focus on utility and clarity in all documentation and API messages.
- **Terminology:** Use standard financial and technical terms (e.g., "Transaction," "Receipt," "Status," "Endpoint") consistently.

## Technical Standards
- **API Design:** RESTful principles with clear resource naming and HTTP status usage.
- **Error Handling:** Structured error responses providing actionable feedback.
- **Code Quality:** Adhere to .NET standard naming conventions and best practices.

## Visual & UX Guidelines (If applicable)
- **Simplicity:** Prioritize ease of use and quick access to core functions.
- **Feedback:** Provide immediate visual feedback for upload status and background processing states.
